MS-DOS (Microsoft Disk Operating System)

üìö About This Guide:

This guide covers MS-DOS commands and concepts, including both original MS-DOS (1981-1995) and modern Windows Command Prompt commands that evolved from DOS. We've clearly marked which features are Windows-specific vs. original DOS.

MS-DOS (acronym for Microsoft Disk Operating System) is an operating system for x86-based personal computers mostly developed by Microsoft. Collectively, MS-DOS, its rebranding as IBM PC DOS, and some operating systems attempting to be compatible with MS-DOS, are sometimes referred to as "DOS" (which is also the generic acronym for disk operating system). MS-DOS was the main operating system for IBM PC compatible personal computers during the 1980s and the early 1990s, when it was gradually superseded by operating systems offering a graphical user interface (GUI), in various generations of the graphical Microsoft Windows operating system.

Historical Timeline

  • 1981: MS-DOS 1.0 released with IBM PC
  • 1982: MS-DOS 1.1 - added support for double-sided disks
  • 1983: MS-DOS 2.0 - introduced subdirectories and hard disk support
  • 1984: MS-DOS 3.0 - added network support
  • 1987: MS-DOS 3.3 - improved disk partitioning
  • 1988: MS-DOS 4.0 - introduced graphical shell
  • 1991: MS-DOS 5.0 - memory management improvements
  • 1993: MS-DOS 6.0 - added MOVE, DELTREE, and disk compression
  • 1994: MS-DOS 6.22 - final standalone version
  • 1995+: DOS integrated into Windows 95 and later

Compatibility and Modern Usage

üí° Modern Usage:
  • Windows Command Prompt (cmd.exe): Most MS-DOS commands still work in Windows Command Prompt
  • PowerShell: Many DOS commands work, but PowerShell has its own enhanced cmdlets
  • DOSBox: Emulator for running original MS-DOS and DOS games
  • FreeDOS: Open-source DOS-compatible operating system
‚ö†Ô∏è Key Differences:
  • Original MS-DOS: Single-user, single-tasking, 16-bit system
  • Windows Command Prompt: Multi-user, multi-tasking, runs on modern Windows
  • File System: Original DOS used FAT12/FAT16, modern systems use NTFS/FAT32
  • Memory: DOS limited to 640KB base memory, Windows has no such limits

Files and Directories

In a computer, data is stored in files. When you run a program, MS-DOS processes the data stored in the file and passes it to the system.

In MS-DOS, a file can be up to the maximum size supported by the file system (typically limited by available disk space), however the file name is more restricted, it can only have a maximum length of 8 characters plus 3 for the extension (known as the 8.3 format).

Furthermore, the files can only contain the following characters:

  • letters A to Z
  • numbers 0 to 9
  • the following special characters: $ ~ ! # % & ‚Äì { } ( ) @ ' _ ^

Thus, filenames must not contain:

  • spaces
  • commas
  • backslash
  • dot (apart from the dot that separates the name and the extension)

Directories

A directory is a location for storing files on your computer. Directories are found in a hierarchical file system, such as Linux, MS-DOS, OS/2, and Unix.

The tree command output shows all the local and subdirectories (e.g., the "big" directory in the cdn directory). When looking at this overview, the C: drive is considered the current directory and root directory because there is nothing beneath it and you can't go back any further. If you are using an operating system with multiple user accounts, the directory may also be referred to as a home directory.

MS-DOS Commands

‚ö†Ô∏è Important Note:

This guide covers both pure MS-DOS commands (from the 1980s-1990s) and modern Windows Command Prompt commands that evolved from DOS. Some commands and features shown here may not work in original MS-DOS systems but are available in Windows Command Prompt (cmd.exe). Where applicable, we'll note which commands are Windows-specific.

1. Internal Commands

Internal commands are loaded automatically into memory.

Command Purpose Syntax
CLS Clears the display screen. C:\> cls
DIR Displays a list of directories and files. C:\> dir
DATE Displays and allows changing the system date. C:\> date
TIME Displays and allows changing the system time. C:\> time
VER Displays the MS-DOS version. C:\> ver
VOL Displays the disk volume label and serial number. C:\> vol [drive:]
DRIVE: Switches to the specified drive. C:\> D: or C:\> A:
COPY CON Creates a new file from keyboard input. C:\> copy con filename.extension
TYPE Displays the contents of a file. C:\> type path
REN Renames a file. C:\> ren oldfilename newfilename
DEL Deletes one or more files. C:\> del filename
MD Creates a new directory. C:\> md directory
CD Changes the current directory. C:\> cd directory_name
RD Removes an empty directory. C:\> rd directory
COPY Copies a file from one location to another. C:\> copy source_path target_path

2. External Commands

External commands require a separate file (e.g., .exe, .bat) to be present on the disk to run. Note: Some commands listed here were introduced in later DOS versions or Windows.

Command Purpose Syntax
CHKDSK Checks a disk and displays a status report. C:\> chkdsk drive_name
DISKCOPY Makes an exact duplicate of a floppy disk. C:\> diskcopy <source> <destination>
FORMAT Erases and prepares a disk for use. C:\> format drive_name
LABEL Views or changes a disk's volume label. C:\> label drive_name
SCANDISK Checks a disk for errors and attempts to repair them. (MS-DOS 6.2+) C:\> scandisk <drive_names>
MOVE Moves files from one place to another. (Available in MS-DOS 6.0+) C:\> move <source> <target>
XCOPY Extended copy with directory support and advanced options. (MS-DOS 3.2+) C:\> xcopy <source> <destination> [options]
REPLACE Replaces files in destination directory with files from source. (MS-DOS 3.2+) C:\> replace <source> <destination> [options]
ATTRIB Displays or changes file attributes (read-only, hidden, system, archive). C:\> attrib [+|-][R|H|S|A] <filename>
PRINT Prints a text file to a line printer. C:\> print <filename>
TREE Displays the directory structure in a graphical tree. C:\> tree path
DELTREE Deletes a directory and all of its files and subdirectories. (MS-DOS 6.0+) C:\> deltree path

Advanced File Copy Operations

XCOPY Command - Extended Copy

XCOPY is much more powerful than the basic COPY command. It can copy entire directory structures and provides many useful options.

Basic Syntax:

C:\> xcopy source destination [options]

Common XCOPY Options:

/S - Copy directories and subdirectories (except empty ones) /E - Copy directories and subdirectories (including empty ones) /V - Verify each copied file /Y - Suppress prompting to confirm overwriting /-Y - Prompt to confirm overwriting /D - Copy only files that are newer than destination /A - Copy only files with archive attribute set /M - Copy only files with archive attribute, then turn off archive bit /H - Copy hidden and system files too /R - Copy read-only files /K - Copy attributes (normally XCOPY resets read-only attribute) /C - Continue copying even if errors occur /I - Assume destination is a directory if copying multiple files /Q - Quiet mode (don't display file names while copying) /F - Display full source and destination file names while copying /L - List files that would be copied (doesn't actually copy)

XCOPY Examples:

// Copy all subdirectories C:\> xcopy C:\Documents D:\Backup /S // Copy all .txt files without prompting C:\> xcopy *.txt D:\TextFiles /Y // Copy entire directory tree C:\> xcopy C:\Project D:\Project /E /Y // Copy only newer files C:\> xcopy source dest /D // Full backup with verification C:\> xcopy /S /E /V /Y C:\Data D:\Backup

ATTRIB Command - File Attributes

The ATTRIB command lets you view and modify file attributes (properties).

File Attributes:

R = Read-only (cannot be modified) H = Hidden (not visible in normal directory listings) S = System file A = Archive (file has been modified since last backup)

ATTRIB Examples:

// View and modify file attributes C:\> attrib filename.txt ; Show current attributes C:\> attrib +R filename.txt ; Make file read-only C:\> attrib -R filename.txt ; Remove read-only attribute // Hide and unhide files C:\> attrib +H secret.txt ; Hide a file C:\> attrib -H secret.txt ; Unhide a file // Work with multiple files C:\> attrib *.* /S ; Show attributes of all files in subdirectories

REPLACE Command

The REPLACE command selectively replaces files in the destination with files from the source.

REPLACE Options:

/A - Add files that don't exist in destination /P - Prompt before replacing each file /R - Replace read-only files too /S - Replace files in subdirectories /W - Wait for disk insertion before starting

REPLACE Examples:

// Replace matching files C:\> replace C:\New\*.* C:\Old // Add files that don't exist in destination C:\> replace source dest /A // Replace DLL files in subdirectories C:\> replace *.dll C:\Windows /S

Batch Files

A batch file is a text file that contains a sequence of commands for a computer operating system. It's called a batch file because it bundles into a single file a set of commands that would otherwise have to be presented to the system interactively one at a time. A batch file is usually created for command sequences for which a user has a repeated need. In the Disk Operating System (DOS), a batch file has the file name extension ".BAT".

Key Features of Batch Files:

  • Automation: Execute multiple commands in sequence
  • Efficiency: Reduce repetitive typing
  • File Extension: Must use .BAT extension
  • Text-based: Can be created with any text editor

Creating and Running Batch Files

Example 1: Simple Backup Batch File (backup.bat)
@echo off cls echo Creating backup of important files... md c:\backup copy c:\documents\*.txt c:\backup\ copy c:\documents\*.doc c:\backup\ echo Backup completed! pause
Example 2: System Cleanup Batch File (cleanup.bat)
@echo off cls echo Cleaning temporary files... del c:\temp\*.tmp del c:\temp\*.bak del c:\windows\temp\*.* echo Cleanup completed! chkdsk c: /f echo System check completed! pause
Example 3: Directory Setup Batch File (setup.bat)
@echo off echo Setting up project directories... md c:\project md c:\project\source md c:\project\backup md c:\project\docs cd c:\project echo Directory structure created! tree pause
To run a batch file:
C:\> backup.bat ; Runs the backup.bat file C:\> cleanup ; Runs cleanup.bat (extension optional)

  • Use rem or :: for comments in batch files
